2. Game Vision (High-Level Canon)

(Summarized from Phase 1 documents — always reference these when creating content.)

Gameplay Identity

A slow, atmospheric, progression-driven online RPG where:

The world is weathered, mythic, and grounded.

Progression is slow but rewarding, driven by skilling, exploration, and mastery.

Combat is deliberate, readable, and grounded, using a hybrid tick + cooldown model.

All core content is solo-completable, with co-op as an optional enhancement.

Exploration, lore discovery, and systemic mastery define the long-term experience.

World Fantasy Statement

(From Phase 1.1 Core Vision)
A fractured realm once guided by scholars and kings, now slipping into forgetfulness.
Old powers — arcane, buried, forbidden — seep upward through ruins, shrines, and whispered rural traditions.
You are not chosen — just a spark in a world where memory clings like moss.

Design Pillars (Finalized in Phase 1.2)

Solo-Friendly Play with Optional Cooperation

Slow, Satisfying Progression via Exploration & Mastery

Storytelling Through Questlines + Environmental Discovery

Assistants must ensure all design aligns with these pillars.

3. Target Player & Tone
Audience

Players who enjoy:

Long-term skilling

Exploration and world discovery

OSRS-like progression pacing

TES-like narrative immersion

Building personal goals

Light co-op without MMO role enforcement

World Tone

Grounded mythic fantasy

Quiet decay + hidden wonder

Magic is rare, risky, mysterious

Light and hope still exist through progress, knowledge, and connection

The world is indifferent, not hostile

Assistant Tone

Clear, practical, production-minded

No unnecessary flourish

Lore writing: atmospheric but restrained

Always prioritize usefulness to a solo developer

4. Core Design Principles
Solo-Dev Friendly

Fewer systems → deeper systems

Identify complexity risks early

Recommend MVP-first, extensible second

Modular & Extensible

Use schemas, templates, and data-driven logic

Keep mechanics tunable

Structure content for future expansion

Player Clarity

Minimize hidden mechanics

Avoid spreadsheet-heavy play

Ensure readable combat and UI flow

Online-First Thinking

Assume co-op minimum

Respect server-authoritative principles

Favor deterministic and predictable systems

5. Technical & Implementation Context

Assume Unity or Godot unless told otherwise

Use engine-agnostic pseudocode unless the user requests C# or GDScript

Networking is initially host-authoritative → scalable to dedicated servers

Persistence uses JSON schemas defined in Phase 1.4 Technical Foundation

Keep code modular, readable, and production-oriented
